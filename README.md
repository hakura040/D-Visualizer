# D+++ Visualizer  
**AI協働型 状態遷移設計・検証ツール**

---

## 概要

D+++ Visualizer は、  
**状態遷移を「図」ではなく「実行可能な仕様」として扱うためのツール**です。

- 状態・遷移・不変条件を構造化して記述
- HACE（Exit / Act / Update / Enter / Check）モデルに基づく実行
- シミュレーション・履歴・ロールバック・テストケース対応
- 参照用の C++ コードコードの雛形（ボイラープレート）作成
- 状態遷移表から状態遷移図の作成（SGV形式でダウンロード可能）

設計段階で状態遷移表で検討したロジックを**実行して確かめる**ことを目的としています。

---

## 想定している使い方・ユーザー

- 状態遷移表を使って設計したいが、  
  仕様の妥当性や説明責任に不安を感じている人
- 実装前に仕様の矛盾・破綻を潰したい人
- AIと一緒に仕様設計を進めたい人

UI操作を覚えるツールではなく、
**思考と設計を支援するための道具**です。

---

## 使い方について

このツールには、  **詳細な操作マニュアルはありません。**

代わりにおすすめしているのは、このHTMLファイルをそのままAI（ChatGPT、Geminiなど）に渡してツールの使い方を質問することです。

このHTMLファイル自体が仕様と設計思想を含んでいるため、AI が文脈を理解してこのツールの使い方を教えてくれます。マニュアルを確認しながら使うよりも、対話型の理解の方が速くスムーズに使えると考えています。

さらに、AIにこのツールを渡すことで状態遷移表の作成をAIに依頼して、このツールでAIが作成した状態遷移表に問題がないか確認を行うことが可能です。

---

## なぜこれを作ったのか

製品設計では、**「なぜそうなっているのか」**が後から問われることが多いです。

手順書や図だけでは、  
- なぜこの遷移が必要なのか
- なぜこの条件が安全なのか
- なぜこの状態に留まれるのか

といった設計意図が、時間とともに失われがちです。

このツールは、  
状態遷移を **実行し、検証できる仕様** として扱うことで、  
設計の理由をそのまま残すために作りました。

また、AIと協働する前提に立つことで、  
状態遷移の記述や検討を人とAIで分担し、  
設計意図を保ったまま詳細化できると考えています。

このツールは  
**「説明責任を取れる速さ」**を重視しています。
